Privacy Policy

Effective date: July 12, 2026

Data Not Collected

DockerBridge does not collect personal data, usage analytics, diagnostics, advertising identifiers, or telemetry.

1. Scope

This policy applies to the DockerBridge macOS application and the website at tecnologica.ar/docker-bridge. DockerBridge is developed by Juan Manuel Mouriz.

2. Information collected by the app

The developer does not receive or collect information from the application. DockerBridge has no user accounts, analytics SDKs, crash-reporting service, advertising, or telemetry.

3. Information stored on your Mac

DockerBridge stores the information needed to provide its functionality locally on your device:

  • Connection profiles and preferences are stored in your user Application Support directory.
  • Passwords are stored in the macOS Keychain and are not written to profile or log files.
  • When key authentication is used, DockerBridge reads the local ED25519 private key selected by the supported configuration. It does not upload or copy that key.
  • Connection logs are stored locally in the configured log directory.
  • Accepted SSH host fingerprints are stored locally to detect new or changed server identities.

4. Network communications

DockerBridge connects directly from your Mac to SSH servers that you configure. It sends the authentication and tunnel traffic required for those connections only to those servers. Traffic is not routed through or made available to the developer.

5. Website data

This website does not use cookies, advertising, analytics, fingerprinting, or third-party tracking scripts. The hosting server may process basic request information such as IP address, date, requested path, and user agent in ordinary technical logs used for security, reliability, and troubleshooting.

6. Third-party software

DockerBridge uses open-source Swift libraries for networking, SSH, and cryptography. These libraries run inside the application and do not provide analytics or advertising services. Their licenses are included with the application and published in the GitHub repository.

7. Sharing and sale of data

The developer does not sell, rent, share, or use personal data for advertising because DockerBridge does not collect that data.

8. Retention and deletion

Local profiles, settings, host fingerprints, credentials, and logs remain under your control. You can remove profiles and credentials from the app, delete logs in Finder, and remove the application support directory when uninstalling DockerBridge.

9. Children

DockerBridge is a technical administration tool and is not directed to children. The application does not knowingly collect information from children or any other users.

10. Changes to this policy

Material changes will be published on this page with an updated effective date. The current version will remain available at this permanent URL.

11. Contact

For privacy questions, contact Juan Manuel Mouriz at jmouriz@gmail.com or open an issue in the GitHub repository.
